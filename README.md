# AI研究实验室官方网站

一个现代化、专业的课题组官方网站，具有完整的用户认证系统，展示研究成果、团队信息并支持与访客互动。

## ✨ 核心功能

- 🏠 **首页展示** - 研究方向和最新论文发表
- 📝 **文章发表** - 在线投稿、写作要求、意见反馈
- 👥 **团队成员** - 多层级团队展示
- 📞 **联系我们** - 联系表单、FAQ、社交媒体
- 🔐 **用户认证** - 完整的注册登录系统（NEW！）

## 🎯 页面功能说明

### 🔐 用户认证系统（NEW！）

#### 功能特性
- ✅ **用户注册** - 新用户创建账号（用户名、邮箱、密码）
- ✅ **用户登录** - 已注册用户安全登录
- ✅ **登出功能** - 一键安全退出
- ✅ **登录保护** - 重要操作需要登录验证
- ✅ **状态显示** - 右上角显示用户名或登录按钮
- ✅ **持久化** - 登录状态自动保存
- ✅ **跨页面同步** - 多标签页状态实时同步

#### 界面位置
- **导航栏右上角** - 登录/注册按钮或用户信息
- **用户下拉菜单** - 个人资料、账号设置、退出登录

#### 需要登录的操作
1. 📝 **文章发表** → 在线投稿 → "提交论文"
2. 💬 **文章发表** → 意见反馈 → "确认提交"  
3. 📧 **联系我们** → 联系表单 → "发送消息"

#### 快速开始
1. 点击右上角"登录 / 注册"按钮
2. 选择"注册"标签创建新账号
3. 填写用户名（≥3字符）、邮箱、密码（≥6字符）
4. 登录后右上角显示用户名
5. 即可使用受保护的功能

### 🔐 认证系统

#### 数据存储
- 使用 `localStorage` 存储用户数据
- 登录状态持久化（关闭浏览器后仍保持）
- 支持多标签页状态同步

#### 安全说明
⚠️ **当前版本为前端演示**，生产环境需要：
1. 后端API验证
2. 密码加密（bcrypt等）
3. HTTPS连接
4. Token认证（JWT）
5. CSRF保护
6. 速率限制

#### 清除数据
在浏览器控制台执行：
```javascript
localStorage.removeItem('users');
localStorage.removeItem('currentUser');
location.reload();
```

#### API示例（后端集成）
```javascript
// 登录API调用示例
async function login(username, password) {
    const response = await fetch('/api/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ username, password })
    });
    return await response.json();
}
```

### 表单提交
目前所有表单提交都是前端处理，显示成功消息。
**后端集成提示**：如需实际发送邮件或保存数据，需要：
1. 创建后端API接口
2. 修改 `contact.js` 中的 `submitContactForm` 函数
3. 修改 `publications.js` 中的 `handleSubmissionForm` 函数
4. 修改 `publications.js` 中的 `handleFeedbackForm` 函数

示例代码（修改 `contact.js`）：
```javascript
// 替换现有的模拟提交代码
fetch('/api/contact', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
    },
    body: JSON.stringify(data)
})
.then(response => response.json())
.then(data => {
    showNotification('消息发送成功！', 'success');
    form.reset();
})
.catch(error => {
    showNotification('消息发送失败，请重试', 'error');
});
```

## 🆘 常见问题

### Q1: 忘记密码怎么办？
A: 当前版本可通过浏览器控制台清除数据重新注册。生产环境应实现"忘记密码"功能。

### Q2: 为什么刷新后还是登录状态？
A: 登录状态保存在localStorage中，除非手动退出或清除浏览器数据。

### Q3: 可以多个标签页同时使用吗？
A: 可以！登录状态会在多个标签页之间自动同步。

### Q4: 密码安全吗？
A: 当前版本仅用于演示，密码明文存储。生产环境必须加密存储。

### Q5: 如何添加新功能？
A: 参考现有代码结构，在对应的JS文件中添加功能，并更新HTML和CSS。
